# Платежи под контролем

Веб-приложение для бухгалтерской аутсорсинговой компании: клиент по персональной
ссылке оставляет заявку на платёж, бухгалтер видит единую очередь по срокам.
Цель — чтобы платежи не терялись в переписке и не срывались сроки.

## Состав

- `index.html` — всё приложение в одном файле (форма клиента + очередь бухгалтера).
- `CONTEXT.md` — «передача дел»: что за проект, что сделано, грабли, роадмап.

## Стек

- **GitHub Pages** — хостинг страницы (публичная ссылка).
- **Supabase** — база данных (таблица `payments`) + хранилище файлов (бакет `files`) + REST API.
- Офлайн-фолбэк через `localStorage`, пока не вписаны ключи Supabase.

## Запуск

Открыть `index.html` в браузере. Персональная ссылка клиента: `АДРЕС/?client=Название компании`.

## Восстановление из бэкапа

Этот коммит — точка отката `v1.0-baseline`. Если работа в папке велась из
git-бандла, восстановить рабочую копию можно так:

```
git clone payments-baseline.bundle restored
cd restored
git checkout v1.0-baseline
```
