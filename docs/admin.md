# Руководство администратора

Администратор (владелец) заводит клиентов, закрепляет за ними бухгалтеров и выдаёт персональные ссылки. Вам доступно всё то же, что бухгалтеру, плюс вкладка **«Клиенты»** и вся очередь целиком.

> Telegram-бот проекта: **@paymentITNIMAX_bot**. В ссылках для перехода используется без `@`: `https://t.me/paymentITNIMAX_bot`.

---

## Вход
1. Откройте https://nikchyv.github.io/payments_new/app/
2. Войдите своим email и паролем.
3. Появятся вкладки **«Очередь платежей»**, **«Клиенты»**, и в шапке — ваше имя с пометкой «владелец».

---

## Завести нового клиента

1. Вкладка **«Клиенты»**.
2. В строке добавления:
   - **Название компании** — например, `ООО «Ромашка»`.
   - **Бухгалтер** — выберите, кто будет вести этого клиента.
3. Нажмите **«Добавить»**.
4. В списке появится карточка клиента с кнопкой **«Скопировать ссылку»**.

### Ссылка клиента
Кнопка «Скопировать ссылку» даёт **ссылку для сайта**:
```
https://nikchyv.github.io/payments_new/app/?t=КОД
```
где `КОД` (после `?t=`) — персональный токен клиента.

**Ссылка для Telegram-бота** формируется из того же кода (имя бота — без `@`):
```
https://t.me/paymentITNIMAX_bot?start=КОД
```
Просто подставьте `КОД` из ссылки сайта.

Пример: для кода `b4a6aedf5f1abe478f4091fa31bcdde6` ссылка бота —
`https://t.me/paymentITNIMAX_bot?start=b4a6aedf5f1abe478f4091fa31bcdde6`

> 🔒 Каждая ссылка персональна и открывает только платежи этого клиента. Передавайте её напрямую клиенту и просите не пересылать посторонним.

---

## Выдать ссылку клиенту
1. Скопируйте ссылку сайта (кнопкой) и при необходимости составьте ссылку бота.
2. Отправьте клиенту любым удобным каналом (см. **шаблоны** ниже).
3. Если клиент хочет пользоваться ботом — он переходит по ссылке бота и нажимает «Старт» (привязка происходит один раз).

---

## Очередь и платежи
Как у бухгалтера (см. [accountant.md](accountant.md)), но вы видите **всех** клиентов и всех бухгалтеров. Фильтр **«Все клиенты»** помогает смотреть по конкретной компании.

---

## Добавить нового бухгалтера

Отдельной кнопки в приложении пока нет — учётка заводится в один приём в панели Supabase:

1. **Supabase → Authentication → Users → Add user** — email и пароль сотрудника.
2. Скопируйте его **UID**.
3. **SQL Editor**, выполните (подставив UID и имя):
   ```sql
   insert into staff (id, name, is_admin) values ('UID', 'Имя сотрудника', false)
   on conflict (id) do nothing;
   ```
4. Передайте сотруднику email/пароль и [руководство бухгалтера](accountant.md).

После этого нового бухгалтера можно выбирать при заведении клиента.

> Полную техническую справку по инфраструктуре см. в файлах `supabase/*.sql` репозитория.

## Шаблоны сообщений для клиентов

Скопируйте, подставьте ссылки и отправьте.

### Шаблон 1 — короткий (сайт)
```
Здравствуйте! Подключаем вас к сервису заявок на оплату.
Ваша персональная ссылка: <ССЫЛКА_САЙТА>

По ней можно оставить заявку на платёж и видеть статус каждого платежа.
Ссылка личная — пожалуйста, не передавайте её другим.
```

### Шаблон 2 — с Telegram-ботом
```
Здравствуйте! Теперь оставлять заявки на оплату можно двумя способами:

🌐 Сайт: <ССЫЛКА_САЙТА>
🤖 Telegram-бот: <ССЫЛКА_БОТА>
   (перейдите и нажмите «Старт» — привязка одноразовая)

Что умеет:
• оставить заявку на оплату (можно приложить фото счёта);
• видеть статус: принята → в работе → оплачено;
• бот сам пришлёт, когда платёж оплачен.

Ссылки персональные — не передавайте посторонним.
```

### Шаблон 3 — инструкция «как оставить заявку»
```
Чтобы оставить заявку на оплату:
1. Откройте ссылку и вкладку «Новая заявка» (или в боте — /new).
2. Укажите получателя, сумму, дату платежа.
3. По желанию приложите фото или PDF счёта.
4. Отправьте — мы увидим заявку сразу.

Статус платежа смотрите во вкладке «Мои платежи» (в боте — /payments).
```
